# ASCII Diagram 표준

## 필수: 기본 ASCII만 사용

**중요**: 다이어그램에는 항상 기본 ASCII 문자만 사용하세요 (최대 호환성).

### ✅ 허용: `+` `-` `|` `^` `v` `<` `>` 및 영숫자 텍스트

### ❌ 금지: Unicode 박스 그리기 문자
- 금지: `┌` `─` `│` `└` `┐` `┘` `├` `┤` `┬` `┴` `┼` `▼` `▲` `►` `◄`
- 이유: 폰트/플랫폼에 따라 일관성 없는 렌더링

## 표준 ASCII 다이어그램 패턴

### 중요: 문자 너비 규칙
**박스의 모든 줄은 정확히 동일한 문자 수를 가져야 합니다 (공백 포함)**

✅ 정확함 (모든 줄 = 67자):
```
+---------------------------------------------------------------+
|                      Component Name                           |
|  Description text here                                        |
+---------------------------------------------------------------+
```

❌ 잘못됨 (일관성 없는 너비):
```
+---------------------------------------------------------------+
|                      Component Name                           |
|  Description text here                                   |
+---------------------------------------------------------------+
```

### 박스 패턴
```
+-----------------------------------------------------+
|                                                     |
|              Calculator Application                 |
|                                                     |
|  Provides basic arithmetic operations for users     |
|  through a web-based interface                      |
|                                                     |
+-----------------------------------------------------+
```

### 중첩된 박스
```
+-------------------------------------------------------+
|              Web Server (PHP Runtime)                 |
|  +-------------------------------------------------+  |
|  |  index.php (Monolithic Application)             |  |
|  |  +-------------------------------------------+  |  |
|  |  |  HTML Template (View Layer)               |  |  |
|  |  |  - Form rendering                         |  |  |
|  |  |  - Result display                         |  |  |
|  |  +-------------------------------------------+  |  |
|  +-------------------------------------------------+  |
+-------------------------------------------------------+
```

### 화살표 및 연결
```
+----------+
|  Source  |
+----------+
     |
     | HTTP POST
     v
+----------+
|  Target  |
+----------+
```

### 가로 흐름
```
+-------+     +-------+     +-------+
| Step1 | --> | Step2 | --> | Step3 |
+-------+     +-------+     +-------+
```

### 레이블이 있는 세로 흐름
```
User Action Flow:
    |
    v
+----------+
|  Input   |
+----------+
    |
    | validates
    v
+----------+
| Process  |
+----------+
    |
    | returns
    v
+----------+
|  Output  |
+----------+
```

## 검증

다이어그램 생성 전:
- [ ] 기본 ASCII만 사용: `+` `-` `|` `^` `v` `<` `>`
- [ ] Unicode 박스 그리기 없음
- [ ] 정렬을 위해 공백 사용 (탭 사용 안 함)
- [ ] 모서리는 `+` 사용
- [ ] **모든 박스 줄이 동일한 문자 너비** (공백 포함하여 문자 수 계산)
- [ ] 테스트: 고정폭 폰트에서 모서리가 수직으로 정렬되는지 확인

## 대안

복잡한 다이어그램의 경우, Mermaid 사용 (`content-validation.md` 참조)
